Technical Architecture

The Eonix Smart PDS is engineered as a layered power architecture that combines fast analog regulation, hardware-enforced protection, and digitally supervised control and telemetry.

All safety-critical behavior is guaranteed in hardware. Software configures operating limits, aggregates telemetry, and provides system-level visibility.

Input Power Architecture

The system accepts a wide input voltage range to support common battery packs, adapters, and industrial power sources.

Input voltage range: 10 V to 25 V

Hardware under-voltage and over-voltage cutoffs are enforced at the input stage. When input limits are violated, the system disconnects loads deterministically to protect downstream electronics.

Protection & Fault Handling

Protection mechanisms operate independently of firmware execution and communication state.

Each channel implements hardware-enforced over-current and short-circuit protection. Faults are isolated at the channel level and do not propagate across the system.

Protection events are latched, timestamped, and reported to the supervisory controller for user visibility and diagnostics.

Power Regulation & Channel Control

Output channels are designed around fast analog control loops to ensure stable voltage and current regulation under dynamic load conditions.

Where applicable, channels support programmable CC/CV behavior. Current limiting is enforced in hardware, not via firmware polling.

Each channel can be individually enabled, disabled, and configured within predefined safe operating boundaries.

Telemetry & Measurement

The system continuously measures voltage and current on the input and on each output channel.

Telemetry data is sampled, aggregated, and made available to the user in real time. Electrical state is never inferred — it is directly measured.

Connectivity & User Interface

A wireless Bluetooth interface provides live access to system telemetry and configuration from a mobile device.

Users can view real-time voltage and current data, enable or disable channels, and receive immediate feedback on system state and fault conditions.

Data Logging, Alerts & Visualization

Telemetry data can be logged over time to capture system behavior during operation, testing, or fault events.

Logged data is visualized as time-series graphs. Protection events and limit violations are explicitly marked on graphs to correlate electrical behavior with system faults.

When a fault or abnormal condition occurs, the system generates immediate alerts to the user through the connected interface.

Detailed electrical limits, channel ratings, protection thresholds, timing characteristics, and block diagrams will be published as the platform completes validation and characterization.
